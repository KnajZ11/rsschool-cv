# HR Planning Tool

Инструмент для планирования загрузки сотрудников по проектам с возможностью управления отпусками, финансовыми расчётами и ежемесячной отчётностью. Разработан в рамках задания Rolling Scopes School.

## Функциональность

- **Управление сотрудниками** – добавление, редактирование, удаление. Автоматический расчёт возраста (18+), обязательные поля.
- **Управление проектами** – создание, редактирование, удаление проектов с бюджетом и ёмкостью.
- **Назначения (capacity)** – привязка сотрудника к проекту с указанием загрузки (0–1.5) и fitness (0–1). Ограничение суммарной загрузки ≤ 1.5, проверка при создании.
- **Финансовые расчёты** – доход, затраты и прибыль по каждому назначению, проекту и общая сумма.
- **Сортировка и фильтрация** – клик по заголовкам таблиц (по убыванию/возрастанию), фильтрация по тексту и должности/компании через модальное окно с чипами.
- **Календарь отпусков** – выбор дней отпуска для каждого сотрудника на текущий месяц. Учтено при расчёте эффективной загрузки.
- **Ежемесячные снимки** – данные хранятся отдельно для каждого месяца (localStorage). Возможность копирования из предыдущего месяца.
- **Навигация по месяцам** – переключение между месяцами с помощью стрелок.
- **Подтверждение действий** – все удаления и изменения с финансовыми последствиями требуют подтверждения.

## Технологический стек

- **TypeScript** – весь код написан на TypeScript (ручная типизация, классы, модули).
- **HTML5 / CSS3** – семантическая разметка, адаптивные стили, CSS Grid для календаря.
- **localStorage** – хранение всех данных на стороне клиента.
- **GitHub Pages** – размещение статической версии.

## Как запустить приложение

### Локально

1. Клонируйте репозиторий и перейдите в папку проекта:

   git clone <https://github.com>
   cd "rsschool-cv/my project"

2. Установите зависимости:

   Убедитесь, что у вас установлен Node.js. Выполните команду в терминале:
  
   npm install  

3. Запустите сервер разработки:

   Вместо простого открытия файла `index.html`, используйте встроенный сервер Vite. Это необходимо для корректной работы модулей TypeScript:

    npm run dev

4. Откройте приложение:

   Перейдите по адресу, указанному в терминале (обычно это [http://localhost:5173](http://localhost:5173)). Приложение будет автоматически обновляться при сохранении изменений в коде.

5. Сборка проекта:

   Чтобы создать оптимизированную версию приложения для размещения на хостинге:

   npm run build

## Специальные заметки о реализации

- **Локальное хранение**: Все данные сериализуются в JSON и хранятся в `localStorage` под ключом `monthlyData`. Структура организована по месяцам: ` { "YYYY-MM": { employees, projects } } `, что обеспечивает независимость данных для разных периодов.
- **Идентификаторы**: Для сущностей (сотрудники, проекты) используются уникальные строковые ID (Base36), генерируемые на основе временной метки, что исключает конфликты при копировании.
- **Валидация**:
  - Проверка возраста сотрудника (строго 18+).
  - Ограничение суммарной загрузки (capacity) одного человека — не более 1.5.
  - Валидация положительных значений для зарплат и бюджетов.

## Особенности финансовых расчётов

- **Revenue (Доход)**: Рассчитывается исходя из «эффективной мощности» сотрудника. Учитывается бюджет проекта, плановая ёмкость и фактическая загрузка всех участников (справедливое распределение бюджета при перегрузе).
- **Cost (Затраты)**: Включает 100% зарплаты за время на проектах и 50% ставки (**Bench**) за неиспользованную часть времени (до 1.0). Если сотрудник занят на 100% и более, «бенч» не начисляется.
- **Vacation Factor**: Коэффициент, снижающий доход и эффективную мощность. Рассчитывается как отношение фактически отработанных будних дней к общему количеству рабочих дней в месяце.

## Используемый технологический стек

- **Vanilla TypeScript**: Приложение написано на чистом TS с использованием классов и модулей без сторонних UI-фреймворков (React/Vue/Angular).
- **Vite**: Инструмент сборки для быстрой разработки и оптимизированного продакшн-бандла.
- **CSS Grid & Flexbox**: Для построения сложной сетки календаря и адаптивности без сторонних библиотек.

- **Vite**: Современный инструмент сборки для обеспечения мгновенного HMR и оптимизации финального бандла.
- **CSS Grid & Flexbox**: Использованы для создания сетки календаря отпусков и адаптивной двухколоночной структуры дашборда.
- **Валидация форм**:
  - Возраст сотрудника не менее 18 лет (проверяется по дате рождения).
  - Суммарная загрузка сотрудника (capacity) не более 1.5.
  - Все числовые поля (бюджет, ёмкость проекта, зарплата) должны быть положительными.
- **Позиционирование всплывающих окон**: Модальные окна центрируются на экране для обеспечения кроссбраузерности и удобства использования на мобильных устройствах.
- **Отпуска**: Дни отпуска хранятся как массив чисел (1–31). Влияют на эффективную загрузку: если день — отпуск, то загрузка в этот день не учитывается при расчёте прибыли.
- **Копирование из предыдущего месяца**: При переключении на новый месяц реализована возможность клонирования структуры команды и проектов из предыдущего периода с сохранением связей.
- **Совместимость**: Приложение поддерживает все современные браузеры (Chrome, Firefox, Edge, Safari). Рекомендуется ширина экрана не менее 800px.
- **Локальное хранение**: Данные сериализуются в JSON и хранятся в `localStorage` под ключом `monthlyData`. Структура организована по месяцам: ` { "YYYY-MM": MonthlyData } `, что позволяет изолировать данные разных периодов.
- **Идентификаторы**: Для сущностей используются уникальные строковые ID (Base36), генерируемые при создании, что исключает коллизии при копировании данных.
- **Финансовые расчёты (строго по ТЗ RSSchool)**:
  - **Revenue (Доход)**: Расчитывается на основе эффективной мощности сотрудника (`capacity * fitness * vacationFactor`). Доход на единицу мощности определяется как отношение бюджета проекта к максимуму между его плановой ёмкостью и фактической суммарной мощностью всех назначенных сотрудников.
  - **Cost (Затраты)**: Включает 100% зарплаты за время, выделенное на проекты, и 50% от зарплаты (**Bench**) за неиспользованную часть ставки (до 1.0). Если суммарная загрузка > 1.0, бенч равен 0.
  - **Profit**: Разница между суммарным доходом и суммарными затратами (включая расходы на простой персонала).
- **Валидация данных**:
  - **Возраст**: Строгая проверка даты рождения (сотрудник должен быть старше 18 лет).
  - **Capacity**: Ограничение суммарной загрузки одного сотрудника (max 1.5) и контроль шага ввода (0.1).
  - **Положительные значения**: Бюджет, зарплата и ёмкость проекта проходят валидацию на значения `> 0`.
- **Отпуска и эффективная мощность**:
  - Дни отпуска (1–31) учитываются через коэффициент `vacationFactor`.
  - Коэффициент равен отношению реально отработанных рабочих дней (исключая выходные и отпуск) к общему количеству рабочих дней в месяце.
- **Копирование данных**: Реализована функция клонирования структуры предыдущего месяца. При копировании создаются новые уникальные ID, обнуляются списки отпусков, но сохраняются связи между сотрудниками и их проектами.
- **Интерфейс**: Реализован на чистом (Vanilla) TypeScript/CSS без использования внешних UI-фреймворков. Сетка календаря и списки адаптированы под разные разрешения экранов (Responsive Design).
